Introduction

ggplot2 is an alternative (and awesome) to base R for data visualization. It is based on The Grammar of Graphics. In this post, we will understand the philosophy behind ggplot2 and learn to build some of the most frequently used plots for visualizing data.

Grammar of Graphics

Grammar of Graphics is a formal system for building plots. The core idea is that any plot can be uniquely described as a combination of

  • a dataset
  • a geom
  • a set of mappings
  • a statistic
  • a position adjustment
  • a coordinate system
  • a faceting scheme

Step 1

Let us build a bar plot from scratch using the mtcars data. We will build the plot incrementally and understand the above components. The first step in any data visualization exercise is to identify the data set. In ggplot2, we can specify the data set using ggplot().

ggplot2::ggplot(data = mtcars)

Step 2

If you compare with the barplot() function from base, you can observe that instead of specifying the variable we are using the whole data set. And most importantly, the ggplot() function does not generate any plots. It just identifies the data that will be used by the other components.

Once you have identified the data set, we need to specify how the data must be visualized. We will do this using geoms. It basically indicate the geometric shapes that must be used to display the data. In our case, we are building bar plot and hence we want the data to be displayed as bars.

There are several geoms and we will explore them one by one. For the time time being, let us use geom_bar() and map the x axis to a variable using mapping. We will use the cyl variable from mtcars.

ggplot2::ggplot(data = mtcars) +
  geom_bar(
    mapping = aes(x = cyl)
  )
